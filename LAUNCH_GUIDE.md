# Подробная инструкция по запуску E2EE на C#

## Требования

- **ОС:** Windows 10/11, macOS 12+, Linux (Ubuntu 20.04+)
- **.NET SDK 8.0** — скачать: https://dotnet.microsoft.com/download/dotnet/8.0
- **4 отдельных окна терминала**

---

## Шаг 0 — Проверка установки .NET

Откройте терминал и выполните:

```
dotnet --version
```

Ожидаемый вывод:
```
8.0.xxx
```

Если команда не найдена — установите .NET 8 SDK по ссылке выше.

---

## Шаг 1 — Распаковка и подготовка

```
# Распакуйте архив E2EE_CSharp.zip в удобную папку
# Затем откройте терминал в папке E2EE_CSharp/

cd E2EE_CSharp
```

---

## Шаг 2 — Установка зависимостей (только для KeyGen)

KeyGen использует библиотеку BouncyCastle для X25519.
Выполните один раз:

```
cd KeyGen
dotnet restore
```

Ожидаемый вывод:
```
  Determining projects to restore...
  Restored KeyGen\KeyGen.csproj (in 1.2s)
```

---

## Шаг 3 — Запуск KeyGen (Терминал 1)

```
cd KeyGen
dotnet run
```

### Пример вывода:

```
=== E2EE Key Generator (X25519 + AES-256-CBC) ===

[Alice] Публичный ключ : A3F8C2D1E4B5067F9A2C3D4E5F6A7B8C9D0E1F2A3B4C5D6E7F8091A2B3C4D5E6
[Bob]   Публичный ключ : 7B8C9D0E1F2A3B4C5D6E7F8091A2B3C4D5E6F7A8B9C0D1E2F3A4B5C6D7E8F9A0

Общий секрет совпадает: True

Сгенерированный общий ключ (hex):
4A7F3C2E1B9D8F6A5C4E3D2B1A9F8E7D6C5B4A3F2E1D0C9B8A7F6E5D4C3B2A1

Введите тестовое сообщение для шифрования: Привет, Боб!

Зашифровано (hex): 9F3A2C1B4E5D6F7A8B9C0D1E2F3A4B5C...
Расшифровано    : Привет, Боб!

HMAC-SHA256: 1A2B3C4D5E6F7A8B9C0D1E2F3A4B5C6D7E8F9A0B1C2D3E4F5A6B7C8D9E0F1A2
HMAC проверка пройдена: True

Скопируйте общий ключ и используйте его в Alice.cs и Bob.cs
```

> ⚠️ **ВАЖНО:** Скопируйте строку общего ключа (hex, 64 символа).
> В примере выше это: `4A7F3C2E1B9D8F6A5C4E3D2B1A9F8E7D6C5B4A3F2E1D0C9B8A7F6E5D4C3B2A1`

---

## Шаг 4 — Запуск Сервера (Терминал 2)

Откройте **второй терминал**, перейдите в папку проекта:

```
cd E2EE_CSharp/Server
dotnet run
```

### Пример вывода:

```
[Server] Запуск ретрансляционного сервера на 127.0.0.1:65432
[Server] Ожидание 2 клиентов...
```

Сервер ждёт подключений. Оставьте это окно открытым.

---

## Шаг 5 — Запуск Боба (Терминал 3)

Откройте **третий терминал**:

```
cd E2EE_CSharp/Bob
dotnet run
```

### Пример вывода:

```
=== Bob (получатель) ===

Введите общий ключ (hex, 64 символа):
```

Вставьте ключ из Шага 3 и нажмите Enter:

```
Введите общий ключ (hex, 64 символа): 4A7F3C2E1B9D8F6A5C4E3D2B1A9F8E7D6C5B4A3F2E1D0C9B8A7F6E5D4C3B2A1

[Bob] Подключён к серверу 127.0.0.1:65432

[Bob] Ожидание сообщений от Алисы...

Bob >
```

В Терминале 2 (Сервер) появится:
```
[Server] Клиент #1 подключился: 127.0.0.1:52341
```

---

## Шаг 6 — Запуск Алисы (Терминал 4)

Откройте **четвёртый терминал**:

```
cd E2EE_CSharp/Alice
dotnet run
```

### Пример вывода:

```
=== Alice (отправитель) ===

Введите общий ключ (hex, 64 символа):
```

Вставьте **тот же** ключ и нажмите Enter:

```
Введите общий ключ (hex, 64 символа): 4A7F3C2E1B9D8F6A5C4E3D2B1A9F8E7D6C5B4A3F2E1D0C9B8A7F6E5D4C3B2A1

[Alice] Подключена к серверу 127.0.0.1:65432

[Alice] Введите сообщение (или 'exit' для выхода):

Alice >
```

В Терминале 2 (Сервер) появится:
```
[Server] Клиент #2 подключился: 127.0.0.1:52342
```

---

## Шаг 7 — Переписка (пример диалога)

### Терминал 4 — Alice пишет сообщение:

```
Alice > Привет, Боб! Это зашифрованное сообщение.
[Alice] ✓ Отправлено (зашифровано, 64 байт)

Alice > Как дела?
[Alice] ✓ Отправлено (зашифровано, 32 байт)
```

### Терминал 3 — Боб получает и отвечает:

```
[Alice] → Привет, Боб! Это зашифрованное сообщение.
Bob > Привет, Алиса! Всё отлично, спасибо!
[Bob] ✓ Отправлено (48 байт)

[Alice] → Как дела?
Bob >
```

### Терминал 4 — Alice получает ответ:

```
[Bob] → Привет, Алиса! Всё отлично, спасибо!
Alice >
```

### Терминал 2 — Сервер видит только зашифрованный трафик:

```
[Server] Ретрансляция пакета 116 байт от клиента #2
[Server] Ретрансляция пакета 84 байт от клиента #1
[Server] Ретрансляция пакета 68 байт от клиента #2
```

> ✅ Сервер передаёт данные, но **не может** их расшифровать — ключ есть только у Алисы и Боба.

---

## Завершение работы

Введите `exit` в терминале Алисы или Боба:

```
Alice > exit
[Alice] Завершение работы.
```

Сервер продолжает работать и готов к новым подключениям.
Для остановки сервера нажмите `Ctrl+C` в его терминале.

---

## Возможные ошибки и решения

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `Connection refused` | Сервер не запущен | Сначала запустите Server, затем Alice/Bob |
| `Неверный ключ` | Ключ неполный или с пробелами | Скопируйте ровно 64 hex-символа без пробелов |
| `HMAC не совпадает` | Разные ключи у Alice и Bob | Убедитесь что оба используют один и тот же ключ |
| `dotnet: not found` | .NET не установлен | Установите .NET 8 SDK с официального сайта |
| `Package restore failed` | Нет интернета для NuGet | Нужен интернет при первом `dotnet restore` |

## Схема взаимодействия

```
┌─────────────────────────────────────────────────────────────┐
│                     KeyGen (разовый запуск)                 │
│   X25519 → Общий секрет → SHA-256 → AES-ключ (32 байта)    │
└─────────────────────────────────────────────────────────────┘
                              │
                    Общий ключ (hex)
                    ┌──────────────┐
                    │              │
              ┌─────▼─────┐  ┌────▼──────┐
              │   Alice   │  │    Bob    │
              │ (Terminal)│  │ (Terminal)│
              └─────┬─────┘  └────┬──────┘
                    │  TCP сокет  │
                    └──────┬──────┘
                    ┌──────▼──────────┐
                    │  Server-Relay   │
                    │  (не знает ключ)│
                    │  просто         │
                    │  пересылает     │
                    │  байты          │
                    └─────────────────┘
```

Каждое сообщение проходит путь:
```
Открытый текст
  → AES-256-CBC (случайный IV)
  → + HMAC-SHA256 (проверка целостности)
  → TCP пакет [длина | HMAC | IV+шифртекст]
  → Сервер (ретрансляция)
  → Проверка HMAC
  → AES-256-CBC расшифровка
  → Открытый текст
```
